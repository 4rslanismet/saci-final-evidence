SACI Final Evidence Portal

SACI is an explainable, graph-based model for measuring SIEM visibility across telemetry, detection controls, MITRE ATT&CK coverage and CTI/MISP enrichment.

This portal brings together the problem statement, methodological approach, laboratory scope, scenario-level score progression and final evidence-closure output in a single publication package. The goal is not only to report a score, but to show transparently which relations, data sources and control chains produce that score.

Canonical publication snapshot

The primary result of the paper is the final package: 99 declared nodes, 171 edge rows, observed=1 in all 171 rows, 6 assets, 12/12 expected asset-log pairs, 25/25 enabled controls, 13/13 in-scope ATT&CK techniques and SACI=100.

The S0-S18 scenarios are a separate historical controlled-validation series. The S8 closure point in that series uses 95 nodes and 173 edges; it is not the same data snapshot as final. The portal does not merge these two layers into a single numeric sequence.

Although final contains no edge rows with observed=0, the LOGSOURCE:Wazuh and MITRE:T1071.001 edge endpoints are not declared in the node table. Therefore, the result is not interpreted as perfect graph integrity.

Problem and motivation

One of the core problems in SOC and SIEM environments is that visibility is often evaluated in fragmented and disconnected ways. The fact that an organization collects logs, generates alerts or enables certain rules does not necessarily mean that critical assets are actually monitored, expected log sources are active, detection controls work and CTI findings become operational alerts.

This work was developed from that common operational gap. In SIEM deployments, questions such as “is data actually arriving?”, “which control covers which technique?”, “is the critical asset really visible?” and “does the CTI enrichment chain reach alert generation?” are often not answerable within a single measurable model. SACI aims to make these questions visible through a graph-based evidence model.

What does it solve?

SACI treats the evaluation of SIEM visibility through only alert counts or dashboard presence as insufficient. A high number of alerts does not prove that all assets and attack techniques that should be monitored are covered. Similarly, a defined rule does not necessarily mean that it has been validated against the relevant log source, asset, MITRE technique and alert-generation chain.

For this reason, the model treats security visibility as an evidence graph problem. Relations are established among asset, log source, detection control, Wazuh rule, MITRE ATT&CK technique, CTI object, platform, score metric and reason-code nodes. These relations are tracked as observed or missing. This makes it possible to show where the visibility chain closes, where it breaks and which score component it affects.

Contribution

The main contribution of this work is an explainable and auditable graph model for measuring SIEM visibility. SACI does not only produce a final score; it also shows which evidence relations produced that score, which relations were closed and which relations remained missing.

The model evaluates log coverage, critical asset coverage, control/alert coverage, MITRE detection coverage, CTI/MISP enrichment coverage and telemetry freshness together. Visibility is therefore assessed through a chain that connects a log source to controls, rules, MITRE techniques, CTI context and freshness.

  • Graph-based modeling: relates distributed security evidence inside a single evidence graph.
  • Explainable score: makes the SACI score readable together with observed and missing graph relations.
  • Scenario-based validation: shows telemetry loss, partial recovery, CTI closure and freshness impact separately.
Evaluation scope

The final evidence package was prepared in a controlled Wazuh-based laboratory environment. Windows Security, Sysmon, PowerShell, Linux authlog/syslog/process records, pfSense syslog events and MISP-based CTI enrichment outputs were connected to the SACI graph model.

The evaluation is presented in two layers. The primary publication result is the final snapshot. The S0-S18 series tests historical controlled scenarios in which the visibility chain breaks and closes again. Critical Sysmon loss, partial recovery, CTI enrichment closure and freshness degradation are modeled in this second layer.

Evidence layers

SACI treats evidence layers as graph relations rather than disconnected tables. An asset is considered visible only when its expected telemetry relation is observed and can be connected to the relevant detection and scoring context.

The CTI layer is interpreted similarly. The final scorer counts lookup, MISP hit, Wazuh alert and MITRE mapping fields as four flags. Since lookup_executed can be initialized from expected_lookup, the first flag is not an independent proof of query execution and the fields alone do not establish a causal event chain.

What it does not claim

SACI does not measure the absolute security level of an environment, does not perform vulnerability scanning and does not claim that every possible attack path has been closed. It answers a narrower and more auditable question: have the expected visibility relations inside the declared scope been confirmed by evidence?

  • SACI does not measure absolute security.
  • SACI is not a vulnerability scanner or an attack-simulation platform.
  • SACI does not claim to cover every possible attack scenario.
  • SACI only answers whether expected visibility relations inside the declared scope are closed.
Reading the final result

In the final state, all 171 edge rows are marked as observed=1 and the SACI score is 100. This indicates that the structured visibility criteria in the declared laboratory scope are closed in that snapshot. Because two edge endpoints are not declared in the node table, the result should be interpreted as edge-row closure, not as perfect endpoint integrity.

The result should be read as an evidence package that answers operational questions such as which relations were missing, how they were closed and which visibility layer affected the score.

Historical scenario score progression

The S0-S18 series is a controlled validation study separate from the final publication snapshot. It shows that the model reports not only a final score, but also the impact of critical telemetry loss, partial recovery, CTI closure and freshness degradation.

All expected visibility relations are observed within the declared final scope. Historical scenarios remain available as methodology evidence rather than as the canonical publication snapshot.
